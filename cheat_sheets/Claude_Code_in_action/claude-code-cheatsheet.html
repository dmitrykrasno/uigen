What is Claude Code?

Terminal-based AI coding assistant. Core strength: Claude (Opus / Sonnet / Haiku) excels at tool use — knowing which tools to call, when, and in clever combinations. Better tool use = handles more complex tasks, better security (no full codebase sent to external servers), and easy extensibility.

Why tool use matters

  • LLMs alone only produce text
  • Tools = read/write files, run commands, web
  • Better tool use → handles harder tasks
  • No full codebase sent to outside servers

Agentic loop

  • Gather context (read files, search)
  • Formulate a plan
  • Take action (edit, run commands)
  • Iterate until done

Context: # Memory & @ Files

# — Memory mode

Type # → write instruction → pick which CLAUDE.md to update. Claude merges intelligently.

  • Fix repeated mistakes permanently
  • Combine with Esc to catch errors fast

@ — File reference

Mention file in prompt or CLAUDE.md → contents auto-included in every request.

  • Prompt: @src/schema.prisma
  • CLAUDE.md: pin DB schema so Claude always knows your data model

Custom Commands

Automate repeated tasks as your own /commands.

Create a command

  1. Create .claude/commands/
  2. Add audit.md with instructions
  3. Restart Claude Code
  4. Run /audit

With arguments

Put $ARGUMENTS in the file:

/write-test src/auth/auth.ts

Any string gets substituted at $ARGUMENTS.

Examples: /audit · /write-test · /security-review

Hooks — All Types

Shell commands that run at specific lifecycle points. Each receives JSON on stdin. Configure via /hooks or edit settings JSON directly.

Hook Cat. When it fires
Pre PreToolUseToolBefore tool runs — can block (exit 2)
Post PostToolUseToolAfter tool runs — provide feedback to Claude
SessionStartSessionSession begins or resumes
SessionEndSessionSession ends
UserPromptSubmitSessionBefore Claude processes user prompt
PreCompactSessionBefore /compact runs (manual or auto)

SubagentStopAgentA subagent (Task) finishes

Debug: PostToolUse matcher "*" + command jq . > post-log.json → inspect exact stdin for any tool call.

Hook Config & Exit Codes

settings.local.json

{
  "PreToolUse": [{
    "matcher": "Read|Grep",
    "hooks": [{ "type": "command",
      "command": "node ./hooks/env_guard.js" }]
  }],
  "PostToolUse": [{
    "matcher": "Write|Edit|MultiEdit",
    "hooks": [{ "type": "command",
      "command": "node ./hooks/tsc.js" }]
  }]
}

Stdin JSON structure

{
  "hook_event_name": "PreToolUse",
  "tool_name": "Read",
  "tool_input": {
    "file_path": "/path/.env"
  }
}

Exit codes

CodeMeaning
0Allow tool call
2Block (Pre only) + stderr → feedback to Claude
Use console.error() not log() — stderr is what Claude sees.

Useful Hook Examples

Pre .env Guard

Block Claude from reading .env

// hooks/env_guard.js
// matcher: "Read|Grep"
const d = JSON.parse(stdin);
const p = d.tool_input?.file_path
       || d.tool_input?.path || '';

if (p.includes('.env')) {
  console.error(
    'Cannot read .env file.');
  process.exit(2);
}

Post TypeScript Checker

Auto-fix type errors after edits

// hooks/tsc.js
// matcher: "Write|Edit|MultiEdit"
const r = exec('tsc --noEmit');

if (r.exitCode !== 0) {
  console.error(r.stdout);
  process.exit(2);
  // Claude gets errors,
  // fixes call sites
}

For untyped languages: run tests instead.

Post Duplicate Detector

Prevent duplicate queries via SDK

// hooks/query_hook.js
// matcher: "Write|Edit" on queries/
import { query } from
  '@anthropic-ai/claude-code';

const r = await query({
  prompt: 'Similar query exist?'
});
if (r.hasDuplicate) {
  console.error(r.message);
  process.exit(2);
}

Watch only key dirs to limit cost.

Claude Code SDK

Use Claude Code programmatically — same tools as terminal.

TypeScript

import { query } from
  '@anthropic-ai/claude-code';

const result = await query({
  prompt: 'Review src/queries
    for duplicate functions',
  options: {
    allowTools: ['Edit']
    // default: read-only
  }
});

Options

  • CLI: claude --print "task"
  • TS: @anthropic-ai/claude-code
  • Python: anthropic-claude-code

Permissions

  • Default: read-only
  • Add write: allowTools: ['Edit']
  • Or via .claude/settings.json

Best for

  • Hooks needing AI reasoning
  • Scripts & pipelines

Pro Tips & Best Practices

Context management

  • /init in every new project
  • Commit CLAUDE.md — team shares context
  • Pin critical files (schema, API contracts) with @
  • /compact between tasks · /clear for new topic

Getting best results

  • Ctrl+V screenshots for UI tasks
  • Plan mode for multi-step / risky changes
  • "ultra-think about…" for hard logic problems

Hooks strategy

  • TypeScript: PostToolUse → tsc --noEmit
  • Any language: PostToolUse → run tests
  • Secrets: PreToolUse guard on .env
  • Watch only key dirs — controls cost

Extensibility

  • Add MCP servers for your stack
  • Build /commands for team workflows
  • GitHub integration → automate PR reviews
  • Stay current: code.claude.com
